# Entwicklungsumgebung einrichten

Für die Mitarbeit am Projekt werden Java 21, Node.js, Docker, Maven und eine IDE benötigt.

## Java 21

Als JDK-Distribution empfehlen wir [Eclipse Temurin 21](https://adoptium.net/) (OpenJDK, kostenlos, von der Adoptium-Community gepflegt).

<Tabs groupId="os">
  <TabItem value="mac" label="macOS">
    Mit [Homebrew](https://brew.sh/):
    ```bash
    brew install --cask temurin@21
    ```
    Installation prüfen:
    ```bash
    java -version
    ```
  </TabItem>
  <TabItem value="win" label="Windows">
    Mit [winget](https://learn.microsoft.com/de-de/windows/package-manager/winget/):
    ```powershell
    winget install EclipseAdoptium.Temurin.21.JDK
    ```
    Danach ein neues Terminal öffnen und prüfen:
    ```powershell
    java -version
    ```
    Alternativ den Installer direkt von [adoptium.net](https://adoptium.net/) herunterladen; dieser setzt `JAVA_HOME` und den `PATH` automatisch.
  </TabItem>
</Tabs>

## Node.js

Für das React-Frontend wird [Node.js](https://nodejs.org/) in der LTS-Version benötigt.

<Tabs groupId="os">
  <TabItem value="mac" label="macOS">
    ```bash
    brew install node
    ```
    Installation prüfen:
    ```bash
    node -v
    npm -v
    ```
  </TabItem>
  <TabItem value="win" label="Windows">
    ```powershell
    winget install OpenJS.NodeJS.LTS
    ```
    Danach ein neues Terminal öffnen und prüfen:
    ```powershell
    node -v
    npm -v
    ```
  </TabItem>
</Tabs>

## Docker

[Docker Desktop](https://www.docker.com/products/docker-desktop/) stellt sowohl die Docker Engine als auch Docker Compose bereit.

<Tabs groupId="os">
  <TabItem value="mac" label="macOS">
    ```bash
    brew install --cask docker
    ```
    Anschließend Docker Desktop aus dem Launchpad starten. Docker läuft im Hintergrund als Menüleisten-App.
  </TabItem>
  <TabItem value="win" label="Windows">
    Installer von [docker.com/products/docker-desktop](https://www.docker.com/products/docker-desktop/) herunterladen und ausführen.

    Docker Desktop benötigt unter Windows das **WSL 2**-Backend (wird beim Setup automatisch eingerichtet, sofern noch nicht vorhanden).
  </TabItem>
</Tabs>

Installation prüfen:
```bash
docker run hello-world
```

## Maven

<Tabs groupId="os">
  <TabItem value="mac" label="macOS">
    ```bash
    brew install maven
    ```
  </TabItem>
  <TabItem value="win" label="Windows">
    1. Das Binary ZIP von [maven.apache.org/download.cgi](https://maven.apache.org/download.cgi) herunterladen und z. B. nach `C:\Program Files\Maven` entpacken.
    2. Systemumgebungsvariable `MAVEN_HOME` auf diesen Ordner setzen.
    3. `%MAVEN_HOME%\bin` zur Umgebungsvariable `Path` hinzufügen.
    4. Ein neues Terminal öffnen und prüfen:
    ```powershell
    mvn -version
    ```
    Alternativ beschreibt [maven.apache.org/install.html](https://maven.apache.org/install.html) die Installation über Chocolatey oder Scoop.
  </TabItem>
</Tabs>

Installation prüfen:
```bash
mvn -version
```

### CIB seven Maven Repository einrichten

:::warning[Pflichtschritt für CIB seven Services]
Das CIB seven Enterprise Repository ist nicht öffentlich zugänglich. Ohne diese Konfiguration schlägt der Maven-Build aller Services fehl, die auf dem [cibseven-template](https://github.com/d135-1r43/cibseven-template) basieren. Zugangsdaten (Kundennummer + Passwort) beim Dozenten erfragen.
:::

Die Zugangsdaten werden einmalig in `~/.m2/settings.xml` hinterlegt:

**In `<servers>` einfügen:**
```xml
<server>
  <id>mvn-cibseven-enterprise</id>
  <username>KUNDENNUMMER</username>
  <password>PASSWORT</password>
</server>
```

**In `<profiles>` einfügen:**
```xml
<profile>
  <id>cibseven-ee</id>
  <repositories>
    <repository>
      <id>mvn-cibseven-enterprise</id>
      <name>CIB seven Enterprise repository</name>
      <url>https://artifacts.cibseven.org/repository/enterprise-group/</url>
    </repository>
  </repositories>
</profile>
```

**In `<activeProfiles>` einfügen:**
```xml
<activeProfile>cibseven-ee</activeProfile>
```

## IDE: IntelliJ IDEA

Wir empfehlen [IntelliJ IDEA](https://www.jetbrains.com/idea/) als Entwicklungsumgebung. Die **Community Edition** ist kostenlos und für dieses Projekt ausreichend. Allerdings fehlen sehr komfortable Features zum Entwickeln mit Spring und Quarkus. Daher sollte die **Ultimate Edition** verwendet werden. 

Als Studierende der THI besteht Anspruch auf die **Ultimate Edition**, kostenlos über das [JetBrains Student Program](https://www.jetbrains.com/community/education/#students).

IntelliJ erkennt Maven-Projekte automatisch beim Öffnen und bringt ein integriertes Terminal, Git-Unterstützung und Debugger für Spring Boot mit. Das JDK kann direkt in IntelliJ unter **File → Project Structure → SDK** verwaltet werden.
